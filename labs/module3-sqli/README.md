# 模块3 挑战:SQL 注入登录绕过

一个最小登录程序 `app.py`。`admin` 的密码是每次启动**随机生成**的强口令——
猜不到、爆破不动。但它构造 SQL 查询的方式有问题。你的任务:**不知道密码,登录为 admin。**

## 运行

```bash
python3 app.py
# 依次输入 username、password
```

正常登录会显示 `Welcome <name>! role=<role>`,失败显示 `Login failed.`。

## 目标

让程序输出 `Welcome admin! role=admin`,而你并不知道 admin 的真实密码。

## 想一想

- 读一读 `app.py`:用户名/密码是怎么拼进 SQL 语句的?(白盒,允许看源码)
- 如果你的输入里带一个**单引号** `'`,拼出来的 SQL 会变成什么样?能不能"逃出"字符串、
  改变查询逻辑?
- SQL 里 `--` 是什么意思?如果用户名填 `admin' -- ` 会发生什么?
- 或者:能不能让 `WHERE` 条件**恒为真**?(注意 `AND` 比 `OR` 优先级高,想清楚注入点放哪个字段)

参考解法与修复在 `.solution/`。

> 这是真实世界最常见、危害最大的 Web 漏洞之一(OWASP 榜首常客)。
> 学完务必看 `.solution/NOTES.md` 里的**修复**——一行参数化查询就能根治。
