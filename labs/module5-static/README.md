# 模块5 挑战A:静态分析找缺陷

`vuln.c` 是一段**故意写错**的 C 代码,里面藏了好几个典型的内存安全缺陷。
这一关不用"跑"程序,而是用**静态分析工具**(不执行、只看源码)把缺陷找出来。

## 工具与用法

```bash
# 1) cppcheck:轻量、基于模式
cppcheck --enable=warning,style vuln.c

# 2) clang 静态分析器(scan-build):基于符号执行/数据流,更深
scan-build gcc -c vuln.c
```

## 目标

- 用上面两个工具,**找出 vuln.c 里所有内存安全缺陷**,说出每个的类型与所在行。
- 想清楚:每个缺陷在运行时会造成什么后果?
- 给出修复。
