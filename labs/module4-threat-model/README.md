# 模块4 练习:对登录服务做威胁建模

这是一道**纸面练习**,没有代码、没有 exploit。产出是一份分析文档。
目的是练"在动手攻防之前,系统地把威胁想清楚"。

## 对象

`labs/module3-sqli/app.py` 那个登录服务(用户名/密码 → 查数据库 → 返回是否登录、角色)。
把它想象成一个真实的 Web 登录后端。

## 任务

写一份威胁模型(可直接写在本目录下一个 `my-threat-model.md`),包含:

1. **数据流与信任边界**
   用文字或 ASCII 画出:外部实体、进程、数据存储、数据流;标出信任边界
   (哪条线把"不可信输入"和"可信内部"分开)。

2. **STRIDE 威胁清单(至少 4 类)**
   对照 S/T/R/I/D/E,各举出本系统里一个具体威胁(不只是定义,要落到这个登录服务上)。

3. **缓解措施**
   对每个威胁,给出对策,并指出它对应哪条 Saltzer & Schroeder 设计原则。

4. **最严重的设计缺陷**
   指出你认为最致命的一个设计层面的问题,并说明为什么。

## 方法提示

- 先问:外部能"碰到"这个系统的哪些点?(攻击面)
- 数据从哪里跨过信任边界进入系统?跨界处做了校验吗?
- 逐个元素套 STRIDE,别漏掉 R(审计/日志)和 D(可用性)。
- 缓解时回看 notes:最小权限、完全仲裁、失败安全默认……

参考答案在 `.solution/REFERENCE.md`(自己先写一版再对照)。
