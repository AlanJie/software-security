# Project 1 Quiz 中文版

> 剧透提醒:本文件包含官方答案和反馈的中文整理。

原始 quiz 要求回答 UMD Software Security Project 1 中提出的问题。为了匹配课程评分器,文本答案不要额外加空格。

## Question 1: 栈溢出的缓冲区

**题型:** math expression

程序里有一个栈上的缓冲区溢出。包含这个被溢出的缓冲区的栈变量叫什么?

**接受答案:**

- `wis`

**反馈:** `wis` 分配在栈上,并且会被 `gets` 调用溢出。

## Question 2: `wis` 的溢出行号

**题型:** numeric

上一题中识别出的缓冲区,运行哪一行代码会导致它溢出? 这里要填行号,不是代码本身。

**接受答案:**

- `62.0`

**反馈:** 原因是 `gets` 会越过目标缓冲区边界继续写入。

## Question 3: 非栈缓冲区

**题型:** math expression

还有另一个漏洞,与第一个栈溢出无关:一个非栈分配的缓冲区可以被越界索引,广义上也可视为缓冲区溢出。哪个变量包含这个缓冲区?

**接受答案:**

- `ptrs`

**反馈:** `l->data` 也可能被溢出,但那依赖于先溢出 `wis`,所以它不是这里要找的独立漏洞。

## Question 4: `ptrs` 的越界行号

**题型:** text match

上一题中识别出的缓冲区,运行哪一行代码会造成越界? 这里要填数字,不是代码本身。

**接受答案:**

- `100`
- `101`
- `102`

**反馈:** 这个越界来自允许索引变量过大。严格说是在第 102 行发生,但第 100 和 101 行也在为这个结果做准备。

## Question 5: `buf` 的地址

**题型:** text match

`main` 函数里的局部变量 `buf` 地址是多少? 可以用十六进制格式,即 `0x` 加 8 位数字,也可以用十进制格式。题目要的是 `buf` 的地址,不是它的内容。

**接受答案:**

- `3221221680`
- `0xbffff130`

**反馈:** 在 `wisdom-alt.c:100` 处下断点,然后打印 `&buf`。

## Question 6: `ptrs` 的地址

**接受答案:**

- `0x0804a0d4`
- `134521044`

**反馈:** 同样在第一个断点处打印 `&ptrs` 即可。

## Question 7: `write_secret` 的地址

**题型:** text match

函数 `write_secret` 的地址是多少? 可以用十六进制或十进制。

**接受答案:**

- `0x8048534`
- `0x08048534`
- `134513972`

**反馈:** 很直接:在 GDB 中打印 `&write_secret`。

## Question 8: `p` 的地址

**题型:** text match

`main` 函数里的局部变量 `p` 地址是多少? 可以用十六进制或十进制格式。

**接受答案:**

- `0xbffff534`
- `3221222708`

**反馈:** 和前面几题一样,在断点处打印变量地址。

## Question 9: 让 `ptrs[s]` 读取 `p`

**题型:** numeric

应该向程序输入什么,才能让 `ptrs[s]` 读取并尝试执行栈变量 `p` 的内容,而不是执行 `ptrs` 指向的缓冲区里的函数指针? 提示:可以对前面已经得到的地址做一点算术。如果成功,程序会执行 `pat_on_back` 函数。填写最小正整数。

**接受答案:**

- `7.71675416E8`

**反馈:** 这是在 GDB 中计算 `(unsigned int)((int *)&p - (int *)&ptrs)` 的结果。不要用 `(unsigned int)&p - (unsigned int)&ptrs`,因为那得到的是字节差,不是指针大小为单位的差值。`ptrs[s]` 的索引单位是指针大小,所以这里需要按 word 计算。

## Question 10: 让 `ptrs[s]` 读取 `buf[64]`

**题型:** numeric

应该输入什么,才能让 `ptrs[s]` 从 `buf` 的第 65 个字节开始读取并尝试执行,也就是从 `buf[64]` 的位置开始? 以 unsigned integer 形式输入答案。

**接受答案:**

- `7.71675175E8`

**反馈:** 在 GDB 中使用 `(unsigned int)((int *)&buf[64] - (int *)&ptrs)`。过程和上一题一样,只是目标起点不同。

## Question 11: `write_secret` 地址的小端表示

**题型:** text match

下面这段输入会通过越界填充 `buf` 的第 65 到 68 个字节。为了让 `ptrs[s]` 执行 `write_secret` 并打印 secret,应该把 `\xEE\xEE\xEE\xEE` 替换成什么? 注意字节序。

```text
771675175\x00AAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAA\xEE\xEE\xEE\xEE
```

**接受答案:**

- `\x34\x85\x04\x08`

**反馈:** `write_secret` 的地址是 `0x08048534`,按小端序写成字节就是上面的形式。

## Question 12: 栈 smashing 的偏移

**题型:** numeric

假设你想溢出 `wis` 变量来做栈 smashing 攻击。可以先输入 `2` 调用 `put_wisdom`,然后输入足够多字节覆盖该函数的返回地址,把它替换为 `write_secret` 的地址。在 `write_secret` 地址之前需要输入多少字节?

**接受答案:**

- `148.0`

**反馈:** 这个数字来自以下计算:

- 128 字节:缓冲区本身
- 12 字节:三个局部变量 `r`、`l`、`v`
- 4 字节:保存的 EBP
- 4 字节:保存的 EDI
- 接下来才是要覆盖的返回地址
